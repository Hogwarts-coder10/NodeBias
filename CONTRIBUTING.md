# 🤝 Contributing to NodeBias

Thank you for your interest in contributing to **NodeBias**!
We welcome contributions that improve fairness, transparency, and usability of AI systems.

---

## 🧠 How You Can Contribute

You can contribute in several ways:

* 🐛 Fix bugs or improve existing features
* ✨ Add new fairness metrics or mitigation techniques
* 📊 Improve data analysis or model evaluation
* 🎨 Enhance frontend/dashboard UI
* 📚 Improve documentation

---

## 🚀 Getting Started

### 1. Fork the Repository

Click the **Fork** button on GitHub and clone your fork:

```bash
git clone https://github.com/your-username/NodeBias.git
cd NodeBias
```

---

### 2. Create a Branch

Use a descriptive branch name:

```bash
git checkout -b feature/your-feature-name
```

Examples:

* `feature/add-equal-opportunity-metric`
* `fix/json-output-structure`

---

### 3. Install Dependencies

```bash
pip install -r requirements.txt
```

---

### 4. Run the Project

```bash
python app.py
```

---

## 🛠 Contribution Guidelines

### ✅ Code Style

* Follow **PEP8** for Python
* Keep functions modular and readable
* Add comments where necessary

---

### ✅ Commit Messages

Use clear and meaningful commit messages:

```bash
feat: add disparate impact calculation
fix: correct bias metric computation
docs: update README
```

---

### ✅ Pull Requests

Before submitting a PR:

* [ ] Code runs without errors
* [ ] Feature is tested
* [ ] Documentation updated (if needed)
* [ ] No unnecessary files added

---

## 📊 Focus Areas

We especially welcome contributions in:

* Fairness metrics (e.g., Equal Opportunity, Calibration)
* Bias mitigation techniques
* Model explainability
* Performance optimization
* UI/UX improvements

---

## ⚠️ Important Notes

* Avoid introducing heavy dependencies without discussion
* Keep the system **lightweight and explainable**
* Maintain the philosophy of **minimal AI usage**

---

## 💬 Reporting Issues

If you find a bug or want to request a feature:

1. Open an Issue
2. Clearly describe:

   * Problem
   * Steps to reproduce
   * Expected behavior

---

## 🙌 Code of Conduct

Be respectful and collaborative.
We aim to build an inclusive and supportive open-source community.
